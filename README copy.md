# 論文關鍵字全自動標註系統

這是一個網頁應用程式，旨在將學術論文的關鍵字標註流程完全自動化。
使用者可以一次上傳多個 PDF 檔案，系統將自動擷取每份文件的標題、作者、年份與摘要，並利用大型語言模型（LLM）為每一篇論文生成核心主題與目標關鍵字，最後將所有結果整理成清晰的報告(csv)。

## ✨ 主要功能

- **批次處理**：支援一次上傳並處理多個 PDF 檔案。
- **自動化擷取**：自動從 PDF 中讀取標題、作者、年份和摘要。
- **AI 產生關鍵字**：整合大型語言模型（透過 API），為每份文件智能生成核心主題與目標。
- **結果呈現與匯出**：在網頁上清晰地列表顯示所有處理結果，並支援一鍵將結果匯出為 CSV 檔案。

## 🚀 使用流程

1.  **上傳檔案**：在「步驟一」區塊，點擊並選擇您想要分析的一個或多個 PDF 檔案。
2.  **輸入 API Key**：在「步驟二」區塊，輸入您的 API 金鑰（本專案預設使用 GitHub Playground 的金鑰格式）。
3.  **開始標記**：點擊「🚀 開始全自動標記」按鈕。
4.  **等待處理**：系統將開始在背景逐一處理所有檔案。由於需要進行多次 AI 分析，這可能會需要數分鐘，請耐心等候。處理期間，按鈕會顯示為「處理中」。
5.  **查看與匯出**：處理完成後，頁面下方會出現結果報告。您可以瀏覽所有論文的標記結果，並可選擇點擊「💾 匯出為 CSV 檔案」將報告儲存下來。

## 🛠️ 安裝與啟動

### 1. 環境設定

首先，請確保您的電腦已安裝 Python 3。接著，安裝本專案所需的相依套件。

```bash
pip install -r requirements.txt
```

### 2. 啟動應用程式

執行以下指令來啟動 Flask 網頁伺服器：

```bash
python app.py
```

### 3. 訪問網頁

伺服器啟動後，在您的瀏覽器中開啟以下網址即可開始使用：

[http://localhost:5000]

## 📂 專案結構

```
.
├── app.py                  # Flask 主應用程式，處理網頁路由與核心邏輯
├── pdf_abstract.py         # 處理 PDF 內容擷取的 Python 模組
├── requirements.txt        
├── README.md 
├── 論文資料標籤_ver2         # LLM下prompt使用的範本 
├── templates/
│   └── index.html          
├── pdf_uploads/            # 儲存使用者上傳的 PDF 暫存檔
├── abstract_output/        # 儲存摘要txt檔以及csv檔
└── output/                 # 儲存匯出的 CSV 結果報告
```

- **`app.py`**: 核心後端伺服器。負責接收前端請求，調用 PDF 處理與 AI 分析功能，並回傳最終結果。
- **`pdf_abstract.py`**: 一個獨立的、功能強大的模組，專門負責從 PDF 檔案中以啟發式規則擷取標題、作者、年份與摘要。
- **`templates/index.html`**: 構成使用者介面的主要檔案，包含 HTML 結構、CSS 樣式與前端 JavaScript 邏輯。
- **`pdf_uploads/`**: 當使用者上傳 PDF 檔案後，檔案會被暫時存放在此資料夾中等待處理。
- **`output/`**: 使用者點擊「匯出 CSV」後，產生的報告檔案會儲存在此。

## 📦 相依套件

- `Flask`：網頁框架。
- `PyMuPDF`：用於讀取與擷取 PDF 內容。
- `OpenAI`：用於與大型語言模型 API 進行互動。
